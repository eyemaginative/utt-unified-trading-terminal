# Contributing to UTT

Thanks for your interest in contributing.

## Workflow

1. Open an issue (bug/feature/proposal)
2. Fork or branch from `main`
3. Make focused changes
4. Add/update docs and tests
5. Open a pull request with clear context

## Commit message format

Use clear, short messages:

- `docs: add build and usage guide`
- `feat: add market data adapter`
- `fix: handle websocket reconnect`

## Pull request checklist

- [ ] Problem and solution described
- [ ] Docs updated
- [ ] Security implications considered
- [ ] Tests added/updated (if applicable)

## Code of conduct

Be respectful, collaborative, and constructive.
